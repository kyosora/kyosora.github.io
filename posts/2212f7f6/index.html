Chrome套件開發教學：自動整理書籤到域名資料夾

發表於2024-02-23|程式設計
|總字數:1.2k|閱讀時間:4分鐘

本教學將引導如何開發一個 Chrome 擴充套件，此套件能將新建的書籤依域名分類到對應資料夾。我們會使用 background.js 來實現這個功能。

前提

擁有基本的 JavaScript 知識。
已安裝 Chrome 瀏覽器。

步驟1：建立擴充套件目錄

首先，在你的電腦上建立一個新的資料夾來存放擴充套件檔案，例如命名為 BookmarkOrganizer。

MV3 注意：Service Worker 是短暫存活（ephemeral）的，Chrome 在閒置後會將它終止，下次事件觸發時重新啟動。因此 folderCreationLocks 和 domainFoldersCache 只在同一次 SW 啟動週期內有效，跨次書籤新增之間快取會歸零。若需跨 SW 週期保存狀態，要改用 chrome.storage.local。

步驟4：載入擴充套件到 Chrome

打開 Chrome 瀏覽器，進入 chrome://extensions/ 頁面。
開啟開發者模式。
點擊「載入未封裝項目」，選擇剛剛建立的 BookmarkOrganizer 資料夾。

步驟5：測試擴充套件

現在，當你建立一個包含 URL 的書籤時，擴充套件會自動將該書籤移動到以該 URL 的域名命名的資料夾內。如果相應的域名資料夾不存在，擴充套件會自動建立一個。

解析 background.js

監聽書籤建立事件：使用 chrome.bookmarks.onCreated.addListener 來監聽書籤的建立事件。
確保主資料夾存在：ensureMainFolderExists 函式會檢查是否存在一個名為「整理資料夾」的資料夾，如果不存在則建立它。建立時未指定 parentId，Chrome 預設會把它放在「其他書籤」(Other Bookmarks) 底下，找不到時往那邊找就對了。
確保域名資料夾存在：ensureDomainFolderExists 函式負責建立和維護以域名命名的資料夾，並將新的書籤移動到相應的資料夾中。

結語

會開發這個是因為日漸增加的書籤，書籤已經突破幾萬筆，看起來很亂，想了想有沒有解決方法，找了 Chrome 套件沒找到能解決這個問題的套件，最後決定自己寫一個。其實是第一次開發 Chrome 擴充套件，此文章當作筆記，沒做到上架的程度。開發中也遇到很多問題，例如一開始書籤一口氣複製進去，域名資料夾重複建立了，發現是非同步原因；也發現根目錄資料夾新增刪除移動都沒反應，查文件才知道根目錄不能做這些操作。
