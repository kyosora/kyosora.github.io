OpenAI 說「聊天已死」，但死的不是聊天——是靠聊天賺錢這件事

發表於2026-06-08|AI產業觀察
|總字數:1.8k|閱讀時間:5分鐘

2026 年 6 月初，OpenAI 一位資深員工對英國《金融時報》丟出一句話：「Chat is dead.」聊天已死。

緊接著的消息是：ChatGPT 要迎來上線以來最大的一次改版。介面重做，未來幾週陸續上線，主動把你往三個地方推——寫程式的 Codex、圖像生成、以及 Canva、Booking.com 這些外部夥伴做的 App。負責 OpenAI 核心產品的 Thibault Sottiaux 描繪的願景是：你有一個屬於自己的 agent，能在工作和生活的每一件事上幫你。

聊天框退到後面，agent 走到台前。這是新聞講的故事。但我把這幾條消息擺在一起讀，看到的是另一個故事。

先別急著相信「聊天已死」

「聊天已死」是個漂亮的句子，漂亮到值得懷疑。

死了嗎？ChatGPT 現在每週有超過 9 億活躍使用者、5000 萬付費訂閱。一個每週被 9 億人打開的介面，很難說它「死了」。對絕大多數人來說，打開一個框、打一句話、拿到答案——這套互動順得很，沒有壞掉，也不需要被搶救。

所以這句話真正的意思，不是聊天這個「介面」死了。我認為它說的是另一件沒講出口的事：靠賣聊天訂閱來養這家公司，這條路走不通了。

差別很大。前者是在講產品體驗，後者是在講商業模式。OpenAI 嘴上談的是前者，動刀的地方卻很大一部分落在後者。

先把話講公道：agent 化本身不是憑空的包裝，它有實打實的技術理由——多步驟的任務、跨好幾個工具的協作、把狀態存下來接著做、企業要的權限控管和稽核軌跡，這些是聊天框天生做不好、得靠 agent 架構才補得上的。所以「方向對不對」和「為什麼是現在、用這種姿態推」是兩個問題，可以有不同的答案。方向我認為是對的；讓我多看兩眼的，是後面那個問題。

OpenAI 真正在解的，是一張不好看的損益表

把改版的時間點和 OpenAI 的處境對齊，事情就清楚了。

據媒體與私募市場資料推估，ChatGPT 每月帶來約 20 億美元收入，但公司整體仍未轉正。它剛完成一輪約 1220 億美元的融資，估值衝到八千五百多億，而 IPO 在前方，目標估值傳聞要破一萬億。另一邊，Anthropic 的估值已經來到九千六百多億美元，咬得很緊。（這些數字多為外界報導與私募市場推算，OpenAI 沒有公開逐項審計財報，看的時候要打個折。）

撐起一萬億估值，需要一個比「月費 20 美元的聊天訂閱」更會賺錢的故事。

agent 超級應用就是那個故事。聊天訂閱是固定月費，用得多用得少都一樣；agent 不一樣——它幫你跑任務、呼叫工具、串接第三方服務，每一個動作都可以是一次計費，而且企業客戶願意為「能幹活的 agent」付的錢，遠比個人訂閱多。新聞裡那句不顯眼的背景——這次改版是「為了搶下利潤豐厚的企業客戶」而做的更大規模重組——才是整件事的重點。

換個說法：聊天介面對使用者很好，對 OpenAI 的損益表不夠好。

超級應用是什麼，對開發者又意味著什麼

撥開「聊天已死」的包裝，超級應用的本質是平台化。

OpenAI 想讓 ChatGPT 從一個「你問它答」的工具，變成一個入口——寫 code、做圖、訂旅館都從這裡進去。Canva、Booking.com 這些夥伴的 App 接進來，變成 ChatGPT 平台上的外掛；而平台手裡握著幾個關鍵槓桿：誰被優先推薦、使用者資料歸誰、抽成怎麼算。這是經典的平台打法，微信當年也是這樣從聊天軟體長成超級應用的。

對開發者，這是機會也是繩索。

機會是：如果你的服務能接進這個生態，理論上能分到那 9 億週活帶來的流量。繩索是：你從此活在它的規則裡。它決定你怎麼被呼叫、抽多少成、哪天改了介面把你的入口埋掉。平台與被平台化，是同一枚硬幣的兩面。每一個考慮接入的團隊，都得先算清楚：我是在借它的流量，還是在幫它把我的使用者養成它的使用者。

「連 prompt 都不用打」是進步還是收編

改版裡還有一句話值得單獨拎出來看：未來甚至要省去使用者輸入提示詞這一步。

聽起來很美——agent 聰明到不用你交代，自己就知道該做什麼。但這背後有個沒明說的轉移：當 agent 替你決定「該怎麼做」，一連串原本你說了算的環節——它怎麼判讀你的意圖、在 Canva 和別家之間選了誰、任務照什麼順序跑——全交給了模型供應商。你不再是那個下指令的人，你變成那個「被服務」的人。

對只是想訂個機票的一般使用者，這是方便。對工程師，我會留個心眼。prompt engineering 這個技能本來就被高估了——真正難的從來不是「會把指令下得漂亮」，而是「知道自己到底要什麼」。AI 把寫 code 的成本壓到趨近於零，瓶頸就移到了「想清楚要什麼」；agent 再聰明，也替你想不清楚你自己都沒想清楚的東西。所以 prompt 會不會消失我不擔心，我比較在意的是別把「想清楚」這件事也一起外包出去。

我會怎麼看這件事

我每天都在用這些工具，包括 OpenAI 的。所以講清楚：我不反對 agent 化，agent 確實是有用的方向，能幫人省掉一堆機械的步驟。

但 agent 的可靠性，老實說還沒跟上它被推出來的速度。它會飄、會自作主張、會在你沒檢查的地方做錯決定——你讓它幫你訂機票，它挑了最便宜但要轉三次機的那班，因為你沒講優先順序，它也沒問。這類問題一個都還沒真正解決，財務壓力卻已經逼著它走上台前。在這個節骨眼上被推著用 agent 跑重要的事，使用者多少是在當白老鼠。

所以「聊天已死」這句話，我會這樣翻譯：聊天沒死，是賣聊天賺不夠的時候到了。下次再有一家公司鄭重告訴你，你用得好好的某個東西「已經死了、該換新的了」，先別急著信。翻到它的帳本那一頁看看——很多時候，需要那個東西去死的，不是你，是它。
