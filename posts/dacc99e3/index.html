SQL中的CASE語法:強大而靈活的條件表達式

CASE 語法讓查詢裡的條件判斷變得直接，根據不同情況回傳不同結果，不需要在應用層多繞一圈。

CASE語法的基本結構

CASE 有兩種寫法：簡單 CASE 和搜索 CASE。

簡單CASE語法

搜索CASE語法

簡單 CASE 是對單一表達式做等值比對，搜索 CASE 則每個 WHEN 都是獨立條件，能處理範圍、NULL、複合邏輯，彈性更大。

CASE語法的特點

  • 靈活：條件邏輯可以很複雜，包含範圍、複合條件、NULL 判斷
  • 可讀：比一堆巢狀 IF 或應用層後處理清楚
  • 通用：SELECT、WHERE、ORDER BY、GROUP BY、UPDATE 都能用

使用CASE進行UPDATE時的注意事項

  1. 效能考量：大型資料表上跑複雜 CASE UPDATE 會有壓力，可以考慮分批處理或確認相關欄位有索引。

  2. 資料一致性：確認 CASE 涵蓋了所有可能的情況，漏掉的條件會落進 ELSE，沒寫 ELSE 則回傳 NULL，悄悄蓋掉原有資料。

  3. 交易管理：大規模或關鍵更新記得包在 Transaction 裡，出錯才能 ROLLBACK。

  4. 先跑 SELECT 預覽：正式 UPDATE 前，把相同的 CASE 邏輯放進 SELECT 看結果是否符合預期，確認後再執行。
