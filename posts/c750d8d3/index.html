解決 C 槽爆炸危機！把 Ollama 模型檔案搬到 D 槽

發表於2024-12-11|AI工具實戰
|總字數:1.2k|閱讀時間:3分鐘

為什麼要移動 Ollama 模型？

Ollama 預設把所有模型檔案丟在 C 槽，下載幾個大模型之後系統碟就開始告急。本教學帶你把模型搬到 D 槽，並在確認搬移成功後刪掉 C 槽的舊資料，真正把空間釋出來。

移動前的準備工作

  1. 確認 D 槽有足夠空間
  2. 在右下角系統匣對 Ollama 圖示按右鍵選「Quit Ollama」，確認背景程序完全停止（直接關視窗不夠，背景程序還在跑）
  3. 若有自訂 Modelfile 或用 ollama create 建立的本地模型，先把 Modelfile 另存備份；官方預建模型可隨時用 ollama pull 重下，不需額外備份

步驟一：建立新的存放位置

  2. 建議用 robocopy 複製。/Z 可斷點續傳；/R:3 /W:5 是失敗重試三次、每次等五秒；/COPY:DAT 只複製資料本體、屬性和時間戳，略過 ACL，一般使用者執行不會遇到權限問題：

複製完成後，D:\Ollama\models 底下應該要有 blobs\ 和 manifests\，路徑長這樣：D:\Ollama\models\blobs、D:\Ollama\models\manifests。佔最多空間的是 blobs\，模型權重都在那裡。

.ollama 目錄底下還有 id_ed25519、id_ed25519.pub、history 這些檔案，那是金鑰和執行紀錄，不是模型，不用搬。

步驟三：設定環境變數

  1. 按下 Windows + R
  2. 輸入 sysdm.cpl 開啟系統內容
  3. 切換到「進階」頁籤
  4. 點選「環境變數」
  5. 在「使用者變數」區域中新增（Ollama 官方文件就是建議改使用者變數）：
    • 變數名稱：OLLAMA_MODELS
    • 變數值：D:\Ollama\models

變數值一定要指到 models 這一層，不是 D:\Ollama。Ollama 會直接把這個路徑當成存放 blobs\ 和 manifests\ 的目錄；如果填成 D:\Ollama，它會去找 D:\Ollama\blobs，結果就是 ollama list 一片空白。

步驟四：驗證設定

  1. 環境變數改完後，在右下角系統匣對 Ollama 圖示按右鍵選「Quit Ollama」，再從開始選單重新開啟，新的環境變數才會被讀到。不放心就直接重開機。
  2. 執行以下指令確認模型可以正常使用： 如果清單列出搬移前的那些模型，就代表 OLLAMA_MODELS 已經指對地方了。

常見問題解答

Q1：移動後模型無法使用怎麼辦？

先確認 OLLAMA_MODELS 是否正確設定為 D:\Ollama\models（不是上一層 D:\Ollama）。確認無誤後若還是不行，直接重開機讓環境變數確實生效。

Q2：要如何確認移動成功？

比對 D:\Ollama\models 和原本 C:\Users\[使用者名稱]\.ollama\models 兩個資料夾的大小，應該要相近（主要看 blobs\ 那一包）。不要拿整個 .ollama 來比，因為裡面還夾著金鑰和執行紀錄這些非模型檔案，比起來反而對不上。最直接的驗證還是跑一次 ollama list，能列出模型就代表成功。

Q3：移動過程中可以取消嗎？

環境變數改完之前都可以取消，回頭繼續就好。最怕的是複製到一半取消：blobs\ 裡的模型檔可能只複製了部分，ollama list 雖然列得出來，但跑模型時會報錯。遇到這種情況，把 D:\Ollama\models\blobs\ 清空，重新用 robocopy 跑一次（指令有 /R:3 重試參數），或是直接 ollama pull <模型名稱> 重新下載也行。

注意事項

  • 移動過程中請勿關機或重新啟動電腦
  • 確保有足夠的磁碟空間
  • 大模型動輒 4–8 GB，搬移前先備份自訂 Modelfile，省去萬一出錯重下的時間
