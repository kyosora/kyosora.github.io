前言

SQLCMD 是隨 SQL Server 一起安裝的命令列工具，日常用途涵蓋執行維護腳本、匯入匯出資料、串接 Windows 排程器自動跑批次作業。比起 SSMS 的 GUI，它的優勢在於可以寫成批次檔、進版控、在沒有桌面環境的伺服器上執行。

變數使用

SQLCMD 腳本變數（$(變數名)）只在腳本內容或 -Q 的查詢字串中做替換，不適用於 -S、-U、-P 這些連線參數——那些位置由 shell 展開，不是 SQLCMD 處理。

用 -v 定義變數，在腳本或查詢字串內引用：

注意：變數值直接做字串替換，沒有任何 quoting 或逸出。若變數值來自外部輸入，攻擊者可以注入任意 SQL。僅在受控的管理腳本中使用 -v，不要把外部輸入直接當作變數值帶入。

若要讓伺服器名稱也可替換，用 shell 變數帶入連線參數（批次檔的 %SERVER%、PowerShell 的 $env:SERVER）；或在腳本內用 :connect $(MyServer) 語法。

錯誤處理

SQLCMD 有兩套獨立的機制，用途不同：

  • SQLCMD 端：:on error exit 讓 sqlcmd 遇到批次錯誤時立即結束並回傳非零結束碼。
  • T-SQL 層：BEGIN TRY / BEGIN CATCH 在伺服器端攔截執行錯誤。

:setvar、:on error 等 SQLCMD 命令在解析階段就無條件處理，跟 T-SQL 執行流程無關——把 :setvar 寫進 BEGIN CATCH 裡不會讓它變成「只在錯誤時執行」。
