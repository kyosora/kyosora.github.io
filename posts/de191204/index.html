你的商品開始在 ChatGPT 裡被賣了：Shopify Agentic Storefronts 技術拆解

發表於2026-03-28|AI電商技術
|總字數:1.5k|閱讀時間:4分鐘|瀏覽量:

3 月 24 日，Shopify 把 560 萬家商店的商品直接塞進了 ChatGPT、Google AI Mode、Microsoft Copilot 和 Gemini 的對話裡。不需要商家安裝任何 app，不需要額外設定，預設就開。

這不是「未來的電商趨勢」。這是上週發生的事。

發生了什麼事

一個消費者在 ChatGPT 裡問「推薦一款適合冬天跑步的防風外套」，ChatGPT 直接列出商品、價格、評價，點擊後跳轉到商家網站完成購買。整個流程中，消費者不需要打開 Google、不需要逛電商平台、不需要比價網站。

數字說話：AI 導流量比 2025 年 1 月成長 7 倍，AI 歸因訂單成長 11 倍。Shopify 一口氣讓 560 萬商家對接 ChatGPT 的 8.8 億月活用戶。

技術架構：三層堆疊

Shopify 不是簡單地把商品目錄丟給 ChatGPT。背後是一套完整的 agentic commerce 架構。

第一層：Shopify Catalog

Shopify 用自家的 LLM 自動分類和標註商品資料。關鍵在於——AI agent 不讀 HTML 描述。如果你的商品規格只寫在商品描述的文字裡，AI 根本看不懂。

商品資料必須結構化地存在 metafields 裡，agent 才能正確解析。

第二層：Universal Commerce Protocol（UCP）

這是 Shopify 和 Google 共同開發的開放協議，讓任何 AI agent 都能跟任何商家完成交易。UCP 不綁定特定的通訊方式，支援三種綁定：

  • REST API：傳統的 HTTPS+JSON，最穩定
  • MCP 綁定：讓 LLM agent 直接呼叫 UCP 工具，不需要額外的 API 封裝
  • Agent-to-Agent（A2A）：agent 之間直接通訊，跳過 HTTP request/response 的開銷

商家端的能力透過 /.well-known/ucp 發布，agent 發請求時帶上自己的 profile URL，商家計算雙方能力的交集，回傳協商結果。這個設計讓不同能力的 agent 和不同規模的商家都能運作。

第三層：結帳流程

Shopify 提供三組 MCP 工具實作 UCP 的核心能力：

工具 功能
Discovery 跨商家搜尋商品
Checkout 建立結帳 session、收集買家資訊、綁定付款、完成交易
Orders 追蹤訂單、物流事件、售後調整

結帳流程不在聊天介面裡完成。手機上是 in-app browser，桌面上是新分頁跳轉到商家網站。Shopify 明確選擇不在 AI 對話裡做結帳——這是信任問題，消費者還沒準備好在聊天視窗裡輸入信用卡號。

付款安全靠 Agent Payments Protocol（AP2）的 Mandates 擴展，agent 呈遞一個加密簽章的授權證明，商家端可以做密碼學驗證，確保使用者確實同意了這筆交易。

對開發者的影響

SEO 的遊戲規則正在改變

傳統 SEO 優化的是 Google 搜尋排名。但當消費者開始在 ChatGPT 裡找商品，你需要優化的是 AI agent 對你商品的理解程度。

具體來說：

  1. 結構化資料變成硬需求：metafields 不再是「有就加分」，而是「沒有就隱形」
  2. Knowledge Base App：商家可以把退換貨政策、FAQ、品牌調性寫進知識庫，agent 回答消費者問題時會引用
  3. 產品分類要精確：Shopify Catalog 的 LLM 會自動分類，但如果你的分類本身就亂，AI 也救不了你

Shopify 的開發者工具

Shopify 推出了 Dev MCP Server，開發者可以在 Cursor、Claude Code 或 shopify.dev 的 dev assistant 裡直接用。AI agent 可以處理完整開發流程：搭鷹架、執行 GraphQL 操作、產生驗證過的程式碼。

不只是 Shopify 商家的事

Shopify 推出了 Agentic Plan，讓沒有用 Shopify 做電商的品牌也能把商品加進 Shopify Catalog，透過同樣的 AI 管道觸及消費者。這意味著 UCP 正在變成一個跨平台的標準，而不只是 Shopify 的內部工具。

我認為值得關注的三件事

第一，「被發現」的方式正在根本性地改變。 過去十年，電商的核心競爭力是在 Google 搜尋結果的第一頁。接下來，核心競爭力可能是讓 AI agent 在對話中推薦你的商品。這兩種「被發現」需要完全不同的優化策略。

第二，UCP 的開放性值得觀察。 Shopify 和 Google 聯手制定開放協議，已經有 20 多家零售商和平台背書。如果 UCP 真的成為標準，它會改變整個電商基礎設施的樣貌。但歷史告訴我們，「開放標準」最後往往變成制定者的競爭優勢。

第三，結帳不在對話裡完成，這個決定很聰明。 在消費者建立起對 AI 對話購物的信任之前，跳轉到商家網站是正確的過渡方案。但長期來看，如果 AP2 的加密授權機制成熟了，in-chat checkout 一定會回來。

現在該做什麼

如果你做電商相關的開發：

  • 確認商品資料是否已經結構化到 metafields，而不是只塞在 HTML 描述裡
  • 研究 UCP 的 /.well-known/ucp 機制，看看你的系統是否能接入
  • 關注 Shopify 的 Dev MCP Server，它可能改變你建立電商應用的方式

如果你做前端：消費者的購物起點正在從瀏覽器搜尋欄移動到 AI 對話框。這對購物頁面的設計、轉換流程的優化，都會產生連鎖反應。

AI 不只是在幫你寫程式碼。它開始幫消費者花錢了。
