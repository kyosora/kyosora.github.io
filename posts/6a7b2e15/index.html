Notion MCP 使用教學：讓 Claude 直接操作你的 Notion 工作區

發表於2025-06-23|AI工具實戰工具與生產力
|總字數:1.3k|閱讀時間:4分鐘

你是不是經常在 Notion 和 Claude 之間來回複製貼上？有了 Notion MCP（Model Context Protocol），Claude 可以直接連接你的 Notion 工作區，幫你建立頁面、整理資料、查詢內容，不用再手動搬資料。

什麼是 Notion MCP？

Notion MCP 讓 Claude 可以直接讀寫你的工作區——不只是讀取筆記，還能新增頁面、查詢資料庫、建立結構化內容。

MCP（Model Context Protocol）是 Claude 和外部工具之間的通訊協定。Notion 官方提供了對應的 MCP Server，串接之後 Claude 就能直接操作你的 Notion，而不是只能在旁邊看著。

開始前的準備

需要準備的東西：

  • Claude Desktop 應用程式（必須是桌面版，Claude.ai 網頁版不支援 MCP）
  • 一個 Notion 帳戶
  • 基本的終端機操作能力
  • 大約 15 分鐘

第一步：建立 Notion Integration

首先要在 Notion 建立一個 Integration，讓 Claude 取得操作工作區的授權。

  1. 前往 Notion Integrations 頁面
    打開 https://www.notion.so/profile/integrations

  2. 建立新的 Integration

    • 點擊「New Integration」
    • 給它取個名字，比如「Claude MCP」
    • 選擇你的工作區
    • 類型保持「Internal」就好

  3. 複製 Integration Token
    建立完成後，你會看到一串以 ntn_ 開頭的金鑰字串（舊版 Token 前綴是 secret_，也可以用）。先複製起來，等一下要貼進設定檔。

  4. 設定權限
    在 Capabilities 頁面中，建議至少開啟：

    • Read content（讀取內容）
    • Update content（更新內容）
    • Insert content（新增內容）

第三步：連接頁面到 Integration

設定完 Integration 後，你還需要告訴 Notion 哪些頁面可以讓 Claude 存取。

  1. 進入你想要 Claude 存取的頁面
  2. 點擊右上角的「⋯」選單
  3. 選擇「Connect to」→ 你的 Integration 名稱

完成這步後，Claude 才能看到並操作這個頁面。沒有授權的頁面，即使 Integration 設定正確，Claude 也讀不到。

第四步：重啟 Claude Desktop

設定完成後，完全關閉 Claude Desktop 應用程式，再重新打開。MCP 設定只在啟動時讀取，沒重啟就不會生效。

如果一切順利，你應該會在 Claude 介面中看到工具圖示，表示 Notion MCP 已連接成功。

實際可以做什麼

串接完成後，直接跟 Claude 說你要做什麼就好。幾個常用的方向：

  • 「幫我在個人筆記頁面建立一篇關於今天學到的 Python 技巧的筆記」
  • 「把我昨天的會議記錄整理成待辦事項，新增到專案管理資料庫」
  • 「幫我找出所有標記為『重要』的筆記」
  • 「建立本月的工作計劃，包含優先級和預計完成時間」

Claude 會直接呼叫 Notion API 完成操作，不需要你自己開 Notion 頁面。

常見問題排除

看不到工具圖示？

  • 確認設定檔格式正確，JSON 語法不容錯，少一個引號或多一個逗號就會整個失效
  • 確認 Integration Token 貼的是完整的值（包含 ntn_ 前綴）
  • 重啟 Claude Desktop

Claude 說找不到頁面？

  • 確認目標頁面已在第三步連接到 Integration
  • 確認 Integration 有開啟 Read content 權限
  • 頁面有父子關係的話，父頁面也要連接，子頁面才能被存取

M1/M2 Mac 遇到問題？

  • 安裝 Rosetta 2：softwareupdate --install-rosetta
  • 確保 Node.js 版本是 18 以上（node -v 確認）

安全性注意事項

  • Integration Token 不要出現在公開的 Git repo 或截圖裡
  • 只把必要的頁面連接到 Integration，不需要全工作區授權
  • 如果只是查詢用途，只開 Read content 就夠，不需要給寫入權限
  • Notion 的 Integration 管理頁面可以看到存取紀錄，定期確認一下

進一步的用法

幾個值得試試的方向：

  • 讓 Claude 替你批次整理筆記格式，比如統一把散落各處的會議記錄整理成固定模板
  • 用 Claude 跨頁面搜尋、彙整資料（Notion 原生搜尋只能找標題，Claude 可以讀內容）
  • 搭配 Claude 的程式碼能力，讓它直接把分析結果寫回 Notion 頁面

設定完之後跟 Claude 說「幫我在 Notion 建立一份今天的工作清單」，它會自己呼叫 API 寫進去，不用再開頁面手動貼。有問題歡迎在留言區討論。

本文基於 2025 年 6 月的 Notion MCP 撰寫，部分介面或設定方式可能隨版本更新調整。
