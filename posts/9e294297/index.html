CSS 偽元素指南

發表於2024-06-09|程式設計
|總字數:868|閱讀時間:3分鐘

偽元素讓你在不碰 HTML 的前提下插入內容、選取特定部分，或者做純裝飾性的事。下面從最常用的 ::before/::after 開始，到幾個容易忽略的偽元素，逐一說明。

什麼是偽元素？

偽元素讓你在文件中插入內容或選取特定部分，不需要修改 HTML 結構。語法是雙冒號 ::（基於向後相容，部分偽元素也接受舊的單冒號 : 寫法）。

CSS 偽元素列表

以下是常見的 CSS 偽元素：

  1. ::before - 在元素內容之前插入生成內容
  2. ::after - 在元素內容之後插入生成內容
  3. ::first-line - 選取元素的第一行
  4. ::first-letter - 選取元素的第一個字母
  5. ::selection - 選取使用者反白的部分
  6. ::placeholder - 選取輸入框的預設提示文字
  7. ::marker - 選取列表項目的標記（點或數字）
  8. ::backdrop - 全螢幕元素底下的背景層

較少使用但在特定情況下很有用的偽元素：

  1. ::cue - 用於 WebVTT 字幕
  2. ::slotted() - 用於 Web Components 中的投影內容

其他偽元素的使用

::first-line 和 ::first-letter

這兩個偽元素常用於排版設計，特別是文章或段落的開頭。

::selection

自定義使用者選取文字時的樣式。

::placeholder

自定義輸入框的預設文字樣式。

::marker

自定義列表項目的標記樣式。

::backdrop

當元素處於全螢幕模式時，為其底下的背景添加樣式。

使用上的限制

  1. ::before 和 ::after 預設是行內元素（display: inline），需要 width/height 時記得改成 block 或 flex。
  2. 偽元素不存在於 DOM 裡，JS 無法直接取得或操作。
  3. img、input（type="text" 等）等替換元素不支援 ::before 和 ::after，因為這類元素的內容由外部資源決定。
  4. 不是所有偽元素都能套在所有元素上，使用前確認相容性。

偽元素的覆蓋範圍比多數人意識到的廣，::before/::after 只是入口，::marker、::placeholder 這類專屬偽元素往往能省掉一層多餘的包裝元素。
