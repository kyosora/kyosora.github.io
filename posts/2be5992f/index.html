Git 基礎教學

這篇文章帶你了解 Git 的基礎使用，包括版本控制的概念、安裝設定，以及日常會用到的指令。

Git 是什麼？

Git 是一個開源的分散式版本控制系統，能有效率地處理大小不一的專案。它的許多概念和設計方式與其他版本控制系統不太一樣，值得花時間理解。

安裝 Git

Git 可以在各種作業系統上執行：

  • Windows：從 Git 官方網站 下載安裝檔。
  • macOS：透過 Homebrew 執行 brew install git。
  • Linux：用套件管理器安裝，例如 Ubuntu 的 sudo apt-get install git。

安裝過程中，預設選項就夠用了，一路 Next 到底即可。

解決 Git Push 失敗

執行 git push 失敗時，錯誤訊息通常能指出原因。以下是幾種常見情況：

原因一：遠端儲存庫有新的提交

遠端（remote repository）有新的提交而本地端尚未同步時，push 會被拒絕。

解法： 先拉取最新變更再推送。

若 pull 過程中出現衝突，手動編輯衝突檔案解決後，重新提交：

原因二：沒有推送權限

對遠端儲存庫沒有寫入權限時，push 會被拒絕。

解法： 聯繫儲存庫的擁有者或管理員，請求取得推送權限。

原因三：分支被保護

部分專案會保護特定分支（如 main），不允許直接推送。

解法： 建立 Pull Request，經過程式碼審查後，由有權限的人合併到目標分支。

原因四：推送到錯誤的分支

分支名稱打錯，或是推送到一個不存在的分支時也會失敗。

解法： 用 git branch -a 確認所有本地與遠端分支名稱，再重新推送。

結語

以上就是 Git 的基礎用法。這些指令在日常開發中很高頻，熟悉之後操作會快很多。
