圖片懶加載技術：提升網站效能的七大方法

發表於2024-03-10|程式設計
|總字數:1.7k|閱讀時間:6分鐘

圖片往往是影響網頁載入速度的主要因素之一。本文整理七種圖片懶加載技術，比較各自的相容性與效能取捨，方便依場景選用。

什麼是圖片懶加載？

圖片懶加載是一種網頁優化技術，延遲載入暫時不可見的圖片。只有當使用者滾動到圖片位置時，圖片才會開始載入，可以顯著減少初始頁面載入時間和頻寬使用。

七種圖片懶加載方法

1. 原生懶加載

HTML Living Standard 定義的 loading 屬性是最簡單的懶加載方式：

支援度：Chrome 77+、Firefox 75+、Safari 16.4+，全球覆蓋率約 95%。不支援的舊版瀏覽器會忽略此屬性，圖片照常載入，無需額外後備處理。

優點：

  • 零程式碼，無需 JavaScript
  • 瀏覽器原生支援，效能佳

缺點：

  • Safari 16.4 以下（含 iOS 16.4 以下）不支援

  • 可能引入不需要的功能

5. 漸進式圖片加載（LQIP）

LQIP（Low Quality Image Placeholder）技術先載入一張體積極小的模糊佔位圖，等高品質圖片下載完成後再切換過去，讓使用者感受到畫面有內容而非空白。

單靠 lazysizes 的基本 data-src 替換並不會產生模糊到清晰的過場效果——那只是延遲載入。要達到真正的漸進式視覺，需要同時引入 ls.blur-up.min.js 外掛，並搭配 CSS 模糊濾鏡：

當 lazysizes 將 data-src 載入並加上 lazyloaded class 後，CSS transition 會把模糊效果平滑移除。

優點：

  • 提供模糊到清晰的視覺過場，比突然出現空白體驗更好
  • 適合大圖或需要高清品質的場景

缺點：

  • 需要準備低品質佔位圖（建議壓縮到 1–5 KB）
  • 需要額外的 CSS 與 lazysizes blur-up 外掛，實施比單純懶加載複雜

  • 需要額外的元件封裝
  • 圖片數量多時，建議共用單一 Observer 實例而非每個元件各自建立

這種方法適合 Vue.js 開發的單頁應用（SPA），懶加載邏輯封裝在元件裡，可在整個應用中重複使用。

實施建議

  1. 選擇適合的方法： 根據網站需求和目標使用者群選擇合適的懶加載技術。
  2. 提供後備方案： 確保在不支援懶加載的瀏覽器中也能正常顯示圖片。
  3. 使用現代圖片格式： 考慮使用 WebP 等現代圖片格式，進一步提升載入速度。
  4. 設定圖片尺寸： 為所有圖片設定適當的 width 和 height 屬性，避免版面跳動（CLS）。
  5. 注意 SEO： 確保重要圖片能被搜尋引擎正確索引。
  6. 框架整合： 使用 Vue.js、React 或 Angular 時，將懶加載邏輯封裝進元件，提高程式碼可重用性。

    結論

七種方法的選用取捨：新專案優先用原生 loading="lazy"，需要精細控制或舊版相容時換 Intersection Observer，只有需要支援極舊瀏覽器才考慮滾動事件版本（記得加 throttle）。背景圖片、LQIP 視覺需求、Vue 元件封裝則按需疊加。
