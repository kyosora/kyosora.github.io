如何在 IIS 上設定 HSTS - 提高網站安全性

今天要和大家分享一個提高網站安全性的重要技術：HTTP Strict Transport Security（HSTS）。這個技術可以說是網站安全的一大利器呢！

最近，專案弱點掃描的時候發現了一個低風險的問題。這個問題就是：

HSTS 是什麼呢？

HSTS 全名是 HTTP Strict Transport Security，是由網際網路工程任務組發布的一種網路安全策略。根據維基百科的說明：

HSTS 的作用是強制瀏覽器使用 HTTPS 與網站進行通信，藉此減少連線被劫持的風險。

簡單來說，HSTS 的主要功能有：

  1. 強制客戶端（如瀏覽器）使用 HTTPS 與伺服器建立連線
  2. 可以有效防禦 SSL 剝離攻擊（這是一種中間人攻擊哦）

聽起來很厲害吧？那麼，我們該如何設定 HSTS 呢？

但是呢，這個方法其實不太正確。因為這樣設定會導致在非 HTTPS 協定下也發送 HSTS 標頭，這是不應該的。

所以呢，要教大家一個更好的方法，就是使用 IIS 的 URL Rewrite 模組來設定。

正確的 HSTS 設定步驟

步驟一：設定 HTTPS 重新導向

  1. 首先，請打開 IIS 管理員介面
  2. 在您的網站下，找到並點選「URL Rewrite」模組
  3. 接著，新增一個輸入規則
  4. 設定如下：
    • 名稱：HTTP to HTTPS redirect
    • 模式：(.*)
    • 條件：{HTTPS}，類型選「符合模式」，模式填「^OFF$」
    • 動作：類型選「重新導向」，重新導向 URL 填「https://{HTTP_HOST}/{R:1}」，記得勾選「附加查詢字串」，重新導向類型選「永久（301）」

步驟二：設定 HSTS 輸出規則

  1. 還是在「URL Rewrite」模組中，這次新增一個輸出規則
  2. 設定如下：
    • 名稱：Add Strict-Transport-Security when HTTPS
    • 變數名稱：RESPONSE_Strict_Transport_Security
    • 模式：.*
    • 條件：{HTTPS}，類型選「符合模式」，模式填「^ON$」
    • 動作：類型選「重寫」，值填「max-age=31536000」


      設定完成後，別忘了重新啟動您的網站，讓新的設定生效喔！

這樣一來，HSTS 的設定就完成啦！是不是感覺很簡單呢？

網路安全真的非常重要。透過這樣的設定，相信大家的網站安全性都能得到很大的提升
